Exercício 2: Asset Advisor Agent+Tools

Desenvolvimento de um Agente orquestrador que compreende intenções de mercado e aciona ativamente uma ferramenta (Tool) de Web Scraping na B3 via Status Invest para obter dados precisos sobre Ações e FIIs.


Reflexões Exigidas (Documentação Analítica)

  • Limitações do Web Scraping: Risco alto de bloqueios com Cloudflare (Erros 403) pela falta de User-Agent e mudanças nas classes DOM HTML.
  • Ticker Inválido: O Agente interpreta as respostas nulas 404 da ferramenta e responde sinteticamente ao usuário que a empresa não existe na bolsa, prevenindo "Error Crashs".
  • Dados Históricos: Como um scraper passivo só tira fotos instantâneas dos cards (preço no dia), para plotar variação histórica dependemos 100% da criação de uma Tool conectada via API real (Ex: Yahoo Finance API).
  • Escalabilidade: Inviável manter Crawler para alto volume. O roadmap ideal troca a ferramenta Crawler por conexões JSON e Redis para Cache das cotações.
